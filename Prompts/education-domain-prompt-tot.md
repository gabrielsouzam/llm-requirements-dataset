Aja como um Engenheiro de Requisitos Sênior experiente. Sua tarefa é analisar o contexto de um projeto de software e, a partir dos exemplos de requisitos fornecidos, gerar a especificação completa dos demais requisitos funcionais que faltam.

# CONTEXTO DO SISTEMA

O projeto é o "Financinhas", uma plataforma online e gamificada de educação financeira destinada ao ambiente escolar. O objetivo é promover uma relação saudável com o dinheiro para crianças e adolescentes através de uma aplicação web.

O sistema utiliza elementos de game design e inclui um módulo de quiz principal chamado "Aventuras do Dinheirinho" como ferramenta de ensino. A plataforma é projetada para ser usada em sala de aula, com perfis distintos para Alunos e Professores.

# EXEMPLOS DE REQUISITOS (30%)

Abaixo estão alguns exemplos de requisitos funcionais do sistema:

## RF001 - Realizar Cadastro de Grupos
O sistema deve permitir o cadastro de grupos de alunos por sala.

## RF004 - Entrar em um grupo
O sistema deve permitir que o aluno participe de um grupo a partir de um código de acesso gerado pelo professor.

## RF008 - Registro de Resultados
O sistema deve registrar o progresso e o resultado de cada Aluno nos jogos e quizzes.

## RF011 - Feedback de Resultado
O sistema deve exibir feedback educativo para respostas corretas e dicas para respostas incorretas, incentivando o aprendizado com feedback imediato.

# INSTRUÇÕES

Siga os seguintes passos (Árvore de Pensamentos - ToT) para derivar os requisitos funcionais faltantes:

Passo 1: Geração de Múltiplos Pensamentos (Exploração de Lacunas)
Analise o Contexto e os Exemplos fornecidos. Gere 3 "linhas de pensamento" (propostas) distintas que explorem as categorias de requisitos que ainda não foram cobertas pelos exemplos.

Proposta 1 (Agente Focado na Gestão de Acesso e Perfis): Gere os cenários faltantes relacionados a como os usuários (Professores e Alunos) se autenticam e gerenciam suas identidades no sistema (ex: cadastro de professores, login, recuperação de senha, visualização de perfis).

Proposta 2 (Agente Focado na Experiência do Aluno): Gere os cenários faltantes relacionados às interações diretas do aluno com o conteúdo de aprendizagem (ex: a personalização do conteúdo, a execução do quiz em si).

Proposta 3 (Agente Focado na Gestão do Professor e Gamificação): Gere os cenários faltantes relacionados às ferramentas do professor para monitorar o progresso e aos elementos de gamificação que completam o ciclo do quiz (ex: visualização de resultados, progressão de dificuldade, ranking).

Passo 2: Autoavaliação dos Pensamentos (Avaliação e Refinamento)
Aja como um "Arquiteto de Solução" e avalie criticamente as 3 propostas. Analise se, em conjunto, elas cobrem todas as funcionalidades implícitas necessárias para um sistema de gestão de quiz em sala de aula (autenticação, personalização, execução e monitoramento). Determine se alguma proposta é redundante ou se falta alguma funcionalidade crítica.

Passo 3: Síntese e Resposta Final (Geração Consolidada)
Com base na sua autoavaliação, combine os melhores e mais relevantes elementos de todas as propostas. Produza a lista final e consolidada dos requisitos funcionais faltantes. A resposta final deve seguir exatamente o mesmo formato de numeração (ex: RFXXX) e descrição usado nos exemplos.

Apresente sua resposta de forma claramente separada por cada passo: Proposta 1, Proposta 2, Proposta 3, Avaliação e, por fim, a Versão Final Consolidada.
