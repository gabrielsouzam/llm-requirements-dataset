Aja como um Engenheiro de Requisitos Sênior experiente. Sua tarefa é analisar o contexto detalhado de um projeto de software e, a partir dos exemplos de requisitos fornecidos, gerar a especificação completa dos demais requisitos funcionais e não-funcionais que faltam.

# CONTEXTO DO SISTEMA

O projeto é o "Cardio Surgery Illustrator", uma solução móvel inovadora para saúde e bem-estar. O objetivo principal é auxiliar estudantes e pacientes no acompanhamento de condições de saúde cardiovascular, conectando informações complexas de maneira intuitiva e acessível. O aplicativo combina funcionalidades de monitoramento de saúde, educação e localização de serviços médicos. Seus principais stakeholders são os pacientes, que desejam acompanhar sua saúde cardíaca, receber recomendações e entender suas condições, e os estudantes, que buscam aprimorar conhecimentos sobre saúde cardiovascular através de materiais e quizzes. O sistema inclui funcionalidades chave como formulários inteligentes para diagnóstico preliminar, visualização interativa em 2D de condições cardíacas, mapas de clínicas, assistente virtual (chatbot), módulos educativos e uma comunidade para troca de experiências.

# EXEMPLOS DE REQUISITOS (30%)

Abaixo estão alguns exemplos de requisitos funcionais e não-funcionais do sistema, derivados das histórias de usuário iniciais:

## RF001 - Realizar Triagem Inteligente
O sistema deve oferecer um formulário inteligente ("Como vai a sua saúde cardiovascular?") que coleta dados sobre hábitos e fatores de risco do paciente para gerar uma avaliação preliminar e direcionar possíveis diagnósticos.

## RF002 - Visualização Interativa 2D
O sistema deve apresentar imagens interativas em 2D que demonstram a evolução de condições cardíacas, permitindo que o usuário interaja para visualizar diferentes fases da progressão da doença no corpo humano.

## RF004 - Localizar Clínicas Próximas
O sistema deve utilizar a geolocalização do usuário para exibir um mapa com clínicas e hospitais próximos, fornecendo informações de contato e endereço para facilitar o acesso ao tratamento.

## RF007 - Assistente Virtual (Chatbot)
O sistema deve disponibilizar um chatbot alimentado por IA para responder dúvidas frequentes sobre saúde e orientar a navegação do usuário dentro do aplicativo de forma automatizada.

## RF016 - Realizar Quiz Educacional
O sistema deve oferecer quizzes interativos sobre saúde cardiovascular voltados para estudantes, fornecendo feedback imediato sobre as respostas para testar e aprimorar o conhecimento.

## RNF01 - Usabilidade Mobile
O sistema deve possuir uma interface otimizada para dispositivos móveis, garantindo navegação fluida e acessibilidade para diferentes perfis de usuários (pacientes e estudantes).

# INSTRUÇÕES

Para garantir uma análise completa e gerar os requisitos funcionais faltantes com alta qualidade, siga esta cadeia de pensamento (Chain-of-Thought) passo a passo:

Passo 1: Análise de Funcionalidades e Stakeholders.
Primeiro, analise os dois principais grupos de usuários (Pacientes e Estudantes) e os objetivos do sistema descritos no Contexto (Monitoramento, Educação, Localização). Liste mentalmente quais necessidades desses grupos ainda não foram atendidas pelos exemplos (ex: O Paciente já tem triagem e mapa, mas como ele acompanha o dia a dia? O Estudante já tem quiz, mas onde ele estuda a teoria?).

Passo 2: Análise de Lacunas (Gap Analysis).
Em seguida, compare os Exemplos (30%) com o escopo completo implícito no Contexto. Identifique as categorias funcionais ausentes:
 Gestão da Saúde: Faltam os relatórios, histórico de análises, sugestões de estilo de vida e notificações (risco/bons hábitos).
 Conteúdo Educativo: Falta a biblioteca de artigos/vídeos, explicação de procedimentos e recursos de acessibilidade (áudio IA).
 Engajamento: Faltam os favoritos, feedback e comunidade.

Passo 3: Geração Final dos Requisitos.
Finalmente, com base nas lacunas detalhadas no Passo 2, formule e apresente a lista final dos Requisitos Funcionais faltantes. Certifique-se de que cada requisito siga exatamente o mesmo formato de numeração (RFXXX) e o nível de detalhe descritivo usado nos exemplos.
